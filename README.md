# REITs Excel Auditor 使用说明

[English Guide](README_EN.md)

REITs Excel Auditor 是一个用于整理 REITs 未审核 Excel 表的小工具。它可以识别不同类型的未审核表，提取需要的字段，并按照标准模板输出为审核格式。

工具支持单个文件处理，也支持文件夹批量处理。所有转换都在本机完成，不需要上传数据到外部服务。

## 主要功能

- 自动识别 1-5 类内置模板未审核表，并转换为对应标准格式。
- 支持手动选择表类型，适合已明确知道表类型的场景。
- 支持上传文件夹，批量识别并逐个输出结果。
- 支持补全信息表，自动补充 REITs 名称、上市日期、公告日期、开始日期和结束日期。
- 支持自定义模板输出，按用户上传模板的字段和格式生成结果。
- 第 4 类产权经营数据可按需额外生成处理版，用于修复和补全主配套资产面积相关字段。

## 快速使用

如果你从源码运行，先安装依赖：

```powershell
python -m pip install -r requirements.txt
```

然后启动软件：

```powershell
python reit_excel_auditor\app.py
```

## 生成 exe 使用

如果你希望把工具打包成可以直接双击运行的 Windows 软件，可以在项目根目录运行打包脚本。

第一次打包前，建议先安装依赖：

```powershell
python -m pip install -r requirements.txt
```

然后执行：

```powershell
.\build_exe.ps1
```

打包完成后，exe 文件会生成在：

```text
dist\REITsExcelAuditor.exe
```

直接双击 `dist\REITsExcelAuditor.exe` 即可打开软件。打开后按界面提示选择输入文件或文件夹、输出位置、转换方式、补全信息表或自定义模板，然后点击“开始转换”。

打包时会生成两个目录：

| 目录 | 说明 |
| --- | --- |
| `dist\` | 最终 exe 文件所在位置。 |
| `build\` | PyInstaller 打包过程使用的中间文件。 |

## 图形界面操作

1. 选择输入文件或输入文件夹。
2. 选择输出位置。
3. 选择转换方式。
4. 如需自动补充基础信息，选择补全信息表。
5. 如需修复第 4 类产权经营数据的主配套资产面积字段，勾选“额外生成第 4 类处理版”。
6. 如需按新模板输出，选择自定义模板。
7. 点击开始转换。
8. 转换完成后，到输出位置查看结果和汇总表。

转换方式说明：

| 方式 | 适用场景 |
| --- | --- |
| 自动识别 | 不确定输入表属于哪一类时使用，软件会按表头字段判断。 |
| 指定表类型 | 已经知道输入表类型时使用。 |
| 自定义模板输出 | 输入表不属于内置 1-5 类模板，但希望按指定模板字段和格式输出时使用。 |

## 输出文件

输出文件会保存在你选择的输出位置。

| 文件 | 说明 |
| --- | --- |
| `原文件名_自动审核.xlsx` | 转换后的标准格式表。 |
| `原文件名_自动审核_处理版.xlsx` | 仅在勾选第 4 类处理版选项且输入为第 4 类时生成。 |
| `自动审核_批量汇总.xlsx` | 批量转换时生成，记录每个文件的识别类型、输出文件、提示信息和失败原因。 |

如果输出表中有空列或缺失字段，汇总表会提示需要人工补充的位置。

## 支持的固定表类型

| 类型 | 输入表 | 输出内容 |
| --- | --- | --- |
| 第 1 类 | 基础资产评估价值表 | 基础资产估值 |
| 第 2 类 | 交通基础设施经营指标表 | 高速经营数据 |
| 第 3 类 | 基础资产财务指标表、招募资产财务指标表 | 资产负债收入成本 |
| 第 4 类 | 基础资产经营指标表 | 产权经营数据，可按需额外生成处理版 |
| 第 5 类 | 能源基础设施经营指标表 | 能源经营数据 |

软件主要根据表头字段识别表类型，不只依赖文件名。

## 第 4 类处理版选项

第 4 类处理版默认不生成。只有在界面勾选“额外生成第 4 类处理版”，或命令行使用 `--property-processed` 时，软件才会输出：

```text
原文件名_自动审核_处理版.xlsx
```

这个文件不是标准汇总格式，而是在原始第 4 类明细表基础上另存一份，并对主配套资产相关字段做辅助修复。适合源表存在面积填错列、合计面积缺失、只有主要资产没有配套资产等情况时使用。

处理逻辑概括：

- 按 `REITs代码` 和 `基础设施项目名称` 分组处理。
- 因为每个项目都必须包含主要资产和配套资产，当某个项目只有主要资产、没有配套资产，会把主要资产行的 `主配套资产名称` 改为 `未分类`。
- 如果同组内 `主配套资产合计的可出租面积(平方米)` 数值不一致，会判断为可能填到了合计列，先移回 `主配套资产单项可出租面积(平方米)`，再重新计算合计。
- 如果主要资产和配套资产的单项面积不同，会把单项面积相加，并把合计写回同组每一行。
- 如果主要资产和配套资产的单项面积完全相同，会判断为可能重复提取，只保留合计面积，清空重复的单项面积。
- `主配套资产单项实际出租面积(平方米)` 与 `主配套资产合计的实际出租面积(平方米)` 使用同样逻辑处理。
- 处理版会统一字体、格式、筛选范围，并取消冻结窗格，打开时选中 `A1`。

如果你的第 4 类源表本身已经核对无误，建议不勾选这个选项，只使用标准输出表即可。

## 补全信息表

补全信息表用于把同一个 REITs 的基础信息自动填入输出表。示例文件在：

```text
examples\metadata_template.xlsx
```

可填写字段：

| 字段 | 说明 |
| --- | --- |
| `REITs代码` | 用于匹配源表中的 REITs 代码，建议填写。 |
| `REITs名称` | 用于补充输出表中的 REITs 名称。 |
| `上市日期` | 用于补充上市日期。 |
| `公告日期` | 用于补充公告日期或 `INFOPUBLDATE`。 |
| `开始日期` | 用于补充 `STARTDATE` 或开始日期。 |
| `结束日期` | 用于补充 `ENDDATE` 或结束日期。 |

字段可以不全。软件会使用已提供的字段，缺失内容会在汇总表中提示。

## 自定义模板输出

自定义模板适合“输入表不是内置 1-5 类表，但你已经有一个希望输出成什么样的 Excel 模板”的场景。软件会读取你上传模板的表头、列顺序和格式，再从输入表中尽量匹配同名或相似字段写入结果。

适合使用的情况：

- 输入表不是内置的 1-5 类固定格式。
- 你只想按自己的模板提取部分字段，不需要内置专属修复逻辑。
- 你希望输出表继承指定模板的列宽、字体、边框、日期格式、百分比格式、数字格式或公式。
- 你有多个同格式输入表，希望放在同一个文件夹里批量套同一个模板。

不建议使用的情况：

- 你需要第 4 类产权经营数据的“处理版”面积修复逻辑。
- 输入文件夹里混放了不同格式的源表。
- 模板表头大量合并单元格，且每个输出列没有自己的字段名。

### 操作步骤

1. 准备一个 `.xlsx` 模板，表头写成你希望输出的字段。
2. 在表头下一行设置示例格式，例如日期、百分比、货币、数字、小数位、字体、边框或公式。
3. 在软件中选择一个输入文件，或选择一个只包含同格式源表的文件夹。
4. 选择“自定义模板文件”。
5. 如需补充 REITs 名称、公告日期、开始日期等信息，可以同时选择补全信息表。
6. 点击“开始转换”。

### 模板怎么设计

- 表头尽量放在同一行，软件会自动寻找前 20 行中最像表头的行。
- 表头不要合并单元格；如果必须合并，建议至少保留每个输出列自己的字段名。
- 表头名称越接近源表字段，匹配越准确。
- 表头下一行会被当作格式参考行，建议在这一行设置好数字格式、日期格式、百分比格式和公式。
- 如果某列希望由公式计算，直接在表头下一行写公式，例如 `=A2+B2`，输出时会自动向下平移为 `=A3+B3`、`=A4+B4`。
- 如果某列只是占位但没有来源字段，也没有公式，输出会留空，并在汇总表里提示。

### 软件会自动处理什么

| 能力 | 说明 |
| --- | --- |
| 字&#8288;段&#8288;顺&#8288;序 | 输出表按模板表头顺序生成。 |
| 字&#8288;段&#8288;匹&#8288;配 | 会尝试完全同名、字段别名、去空格/括号后的匹配、包含关系和相似字段名匹配。 |
| 格&#8288;式&#8288;继&#8288;承 | 会继承模板中的列宽、字体、边框、对齐、筛选、冻结窗格和数字格式。 |
| 日&#8288;期&#8288;处&#8288;理 | 模板列是日期格式，或字段名类似 `公告日期`、`STARTDATE`、`ENDDATE` 时，会尽量转换为日期或 8 位日期数字。 |
| 百&#8288;分&#8288;比&#8288;处&#8288;理 | 模板列是百分比格式，或字段名包含 `率` 时，会按百分比逻辑处理，并保留大于 100% 的真实比例。 |
| 数&#8288;字&#8288;处&#8288;理 | 字段名包含金额、面积、数量、收入、成本、利润、资产、负债等词时，会尽量转换为数字。 |
| 补&#8288;全&#8288;信&#8288;息 | 如果模板需要 `REITs代码`、`REITs名称`、`上市日期`、`公告日期`、`STARTDATE`、`ENDDATE`，但源表没有，软件会尝试用补全信息表按代码填充。 |
| 公&#8288;式&#8288;列 | 如果模板表头下一行有公式，例如 `=A2+B2`，输出时会自动复制到每一行，并按行号平移为 `=A3+B3`、`=A4+B4`。 |

### 常见字段别名

这些写法可以直接被识别，不一定需要手动改字段名：

| 目标字段 | 常见写法 |
| --- | --- |
| `REITs代码` | `ReitsCode`、`代码`、`证券代码`、`基金代码`、`产品代码` |
| `REITs名称` | `REITs简称`、`基金名称`、`基金简称`、`证券简称`、`证券名称`、`产品名称` |
| `公告日期` | `INFOPUBLDATE`、`信息披露日期`、`披露日期` |
| `STARTDATE` | `开始日期`、`报告期开始日期`、`报告期起始日期`、`区间开始日期` |
| `ENDDATE` | `结束日期`、`报告期结束日期`、`报告期截止日期`、`区间结束日期` |

如果你的字段还有其他叫法，可以在 `config\field_aliases.json` 中新增别名。

### 补全信息表的匹配规则

- 优先按源表中的 `REITs代码`、`ReitsCode`、`证券代码`、`基金代码` 或 `产品代码` 匹配补全信息表。
- 如果源表没有代码，而补全信息表只有一行，软件会把这一行作为安全兜底用于当前输入表。
- 如果源表没有代码且补全信息表有多行，软件不会猜测使用哪一行，会在输出中留空并在汇总表提示。
- 补全信息表字段名不必完全固定，`基金简称`、`证券简称`、`信息披露日期`、`报告期开始日期`、`报告期截止日期` 等常见写法都可以识别。

### 文件夹批量使用提醒

使用自定义模板处理文件夹时，请确保文件夹内所有待转换表格式一致。软件会检查每个文件和第一个文件的表头相似度；如果差异较大，会跳过对应文件并在汇总表中提示，避免把不同格式的表硬套到同一个模板里。

自定义模板模式不会执行内置 1-5 类的专属修复逻辑，例如第 4 类处理版面积修复。如果需要第 4 类处理版，请不要使用自定义模板模式，改用普通转换并勾选第 4 类处理版选项。

## 高级配置

`config/` 目录用于维护模板文件名和字段别名。

| 文件 | 作用 |
| --- | --- |
| `config\table_templates.json` | 指定每类固定表使用哪个标准模板文件。 |
| `config\field_aliases.json` | 指定字段别名，让源表字段和目标字段更容易匹配。 |

一般用户不需要修改配置。只有在更换标准模板文件名、或者发现某些字段名称需要新增别名时，才需要调整。

详细说明见：

```text
config\README.md
```

## 常见提示

| 情况 | 处理方式 |
| --- | --- |
| 软件提示缺少字段 | 检查源表是否有对应列，或在输出表中手动补充。 |
| 自动识别失败 | 尝试手动选择表类型，或使用自定义模板输出。 |
| 文件夹批量转换失败部分文件 | 查看 `自动审核_批量汇总.xlsx` 中对应失败原因。 |
| 自定义模板有字段没填上 | 检查模板表头和源表表头是否名称接近，必要时在 `config\field_aliases.json` 增加别名。 |
| 打不开 Excel 文件 | 确认文件没有被 Excel 占用，且不是 `~$` 开头的临时文件。 |
