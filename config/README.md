# 配置文件说明

`config/` 目录用于维护软件的高级匹配规则。大多数用户不需要修改这里；只有在更换标准模板文件名，或者发现某些 Excel 表头叫法软件识别不到时，才需要调整。

修改前建议先复制一份原文件备用。修改后重新打开软件，再用一两个小样本试转，确认无误后再批量处理。

## 文件一览

| 文件 | 什么情况下改 | 作用 |
| --- | --- | --- |
| `table_templates.json` | 更换 `standard_templates\excel_conversion\` 中的 1-5 类标准模板文件名时 | 告诉软件每一类固定表使用哪个标准模板。 |
| `field_aliases.json` | 某些字段名匹配不到时 | 告诉软件“这些不同叫法其实是同一个字段”。 |

年报现金流更新的标准格式模板固定放在 `standard_templates\annual_update\`，一般不需要通过这里配置。

## table_templates.json

这个文件只管理内置 1-5 类固定转换使用的标准模板文件名。

当前类型含义：

| 类型 | 对应输出 |
| --- | --- |
| `valuation` | 第 1 类，基础资产估值 |
| `traffic` | 第 2 类，高速经营数据 |
| `finance` | 第 3 类，资产负债收入成本 |
| `property` | 第 4 类，产权经营数据 |
| `energy` | 第 5 类，能源经营数据 |

注意事项：

- 这里只写文件名，不写完整路径。
- 模板文件必须放在 `standard_templates\excel_conversion\` 目录下。
- 文件名需要包含 `.xlsx` 后缀。
- 不要修改左侧类型名称，例如 `valuation`、`traffic`、`finance`。
- 这个配置不影响“自定义模板输出”，自定义模板是在软件界面里单独选择的。
- 这个配置不影响“年报现金流更新”，年报模板在 `standard_templates\annual_update\`。

## field_aliases.json

这个文件用于字段别名匹配。它的结构是：

```json
{
  "目标字段名": [
    "源表里可能出现的字段名",
    "另一个常见叫法"
  ]
}
```

这个配置同时用于：

- 内置 1-5 类固定转换中的字段匹配。
- 自定义模板输出中的字段匹配。
- 年报更新中部分标准化字段的识别。

新增字段别名的步骤：

1. 找到输出模板中的目标字段名。
2. 在对应数组中加入源表可能出现的字段名。
3. 保存 `field_aliases.json`。
4. 重新打开软件。
5. 用小样本试转并查看汇总表提示。

JSON 格式注意事项：

- JSON 文件不能写注释。
- 字段名必须使用英文双引号。
- 多个字段之间使用英文逗号。
- 数组或对象最后一项后面不要加逗号。
- 修改时只改字段名文本，不要删除外层大括号 `{}`。

## 怎么判断是否需要改配置

建议先不改配置，直接试转一次。如果输出汇总表提示“缺少来源字段”或“自定义模板有字段留空”，再检查源表表头和目标字段是否只是叫法不同。

如果只是叫法不同，就在 `field_aliases.json` 中新增别名。如果源表本身没有这个字段，配置文件无法自动生成内容，仍需要在输出 Excel 中人工补充。
