# 补全信息表使用说明

`metadata_template.xlsx` 是补全信息表示例。它用于在转换时自动补充 REITs 基础信息，例如 REITs 名称、上市日期、公告日期、开始日期和结束日期。

这个文件不是必须使用的。如果源表里已经有完整信息，可以不选择补全信息表。

## 什么时候使用

建议在这些情况下使用补全信息表：

- 源表缺少 `REITs名称` 或 `REITs简称`。
- 源表缺少 `上市日期`、`公告日期`、`开始日期` 或 `结束日期`。
- 自定义模板中需要 `REITs代码`、`REITs名称`、`INFOPUBLDATE`、`STARTDATE`、`ENDDATE` 等字段，但源表没有。
- 同一批表属于同一个或少数几个 REITs，希望统一补充基础信息。

## 模板字段

`metadata_template.xlsx` 默认包含这些字段：

| 字段 | 是否建议填写 | 用途 |
| --- | --- | --- |
| `REITs代码` | 建议填写 | 用于和源表中的 REITs 代码匹配。 |
| `REITs名称` | 可选 | 补充输出表中的 REITs 名称或简称。 |
| `上市日期` | 可选 | 补充上市日期。 |
| `公告日期` | 可选 | 补充公告日期或 `INFOPUBLDATE`。 |
| `开始日期` | 可选 | 补充开始日期或 `STARTDATE`。 |
| `结束日期` | 可选 | 补充结束日期或 `ENDDATE`。 |

字段可以不全。软件只会使用你已经填写的内容，缺少的内容会保留为空，并在转换汇总表中提示。

## 可以识别的常见写法

为了兼容不同来源的表，补全信息表不一定必须使用标准字段名。下面这些常见写法也可以识别：

| 标准字段 | 常见写法 |
| --- | --- |
| `REITs代码` | `ReitsCode`、`代码`、`证券代码`、`基金代码`、`产品代码` |
| `REITs名称` | `REITs简称`、`基金名称`、`基金简称`、`证券简称`、`证券名称`、`产品名称` |
| `上市日期` | `发行日期` |
| `公告日期` | `INFOPUBLDATE`、`信息披露日期`、`披露日期` |
| `开始日期` | `STARTDATE`、`报告期开始日期`、`报告期起始日期`、`区间开始日期` |
| `结束日期` | `ENDDATE`、`报告期结束日期`、`报告期截止日期`、`区间结束日期` |

发布给其他用户使用时，仍建议保留示例模板中的标准字段名，这样最不容易产生歧义。

## 匹配规则

软件会优先用源表中的 `REITs代码`、`ReitsCode`、`证券代码`、`基金代码` 或 `产品代码` 去匹配补全信息表。

如果源表代码带 `.SH` 或 `.SZ` 后缀，软件会尽量兼容匹配。例如 `180301.SZ` 和 `180301` 可以匹配到同一个代码。

特殊情况：

- 如果源表没有代码，且补全信息表只有一行，软件会把这一行作为安全兜底使用。
- 如果源表没有代码，但补全信息表有多行，软件不会猜测使用哪一行，对应字段会留空。
- 一个 REITs 代码建议只保留一行信息，避免后面的记录覆盖前面的记录。

## 日期填写方式

日期可以使用以下任意一种方式：

| 写法 | 示例 |
| --- | --- |
| Excel 日期 | `2026/4/30` |
| 8 位数字 | `20260430` |
| 文本日期 | `2026-04-30` |

建议同一个补全信息表中尽量使用同一种日期写法，方便后续检查。

## 使用步骤

1. 打开 `metadata_template.xlsx`。
2. 按表头填写 REITs 基础信息。
3. 保存文件，文件名可以自行修改。
4. 在软件界面选择输入文件或输入文件夹。
5. 在“补全信息表”位置选择这个文件。
6. 点击“开始转换”。
7. 转换完成后查看输出 Excel 和 `自动审核_批量汇总.xlsx` 中的提示。

## 和自定义模板一起使用

自定义模板输出时，补全信息表也会生效。常见用法是：

- 自定义模板需要 `REITs名称`，但源表只有项目明细，没有 REITs 名称。
- 自定义模板需要 `STARTDATE`、`ENDDATE` 或 `INFOPUBLDATE`，但源表没有日期字段。
- 输入文件夹里每个源表格式相同，但基础信息希望从同一张补全信息表中统一填写。

如果自定义模板里的字段既能从源表匹配到，又能从补全信息表匹配到，软件会优先使用源表中的值；源表为空时再尝试使用补全信息表。

## 小提示

- 不需要的列可以留空，不建议删除表头。
- 如果补全后仍有空字段，先查看汇总表提示，再检查源表代码和补全信息表代码是否一致。
- 如果一批文件对应多个 REITs，补全信息表可以填写多行，每行一个代码。
- 如果一批文件全部属于同一个 REITs，补全信息表只填一行也可以。
