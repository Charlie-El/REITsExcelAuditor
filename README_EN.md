# REITs Excel Auditor

[中文说明](README.md)

REITs Excel Auditor is a Windows desktop tool for converting unaudited REITs Excel workbooks into standardized audit-ready formats. It detects supported workbook types, extracts matching fields, inherits the target template style, and writes cleaned output workbooks.

The tool supports both single-file conversion and folder-based batch conversion. All processing runs locally on your computer, and no workbook data is uploaded to any external service.

## Features

- Detects and converts the five built-in unaudited REITs workbook types.
- Allows manual table-type selection when the input type is already known.
- Supports folder upload for batch conversion.
- Supports a metadata workbook for filling REITs code, REITs name, listing date, announcement date, start date, and end date.
- Supports custom output templates for source workbooks that are not covered by the built-in conversion rules.
- Optionally generates an extra processed version for Type 4 property operation data to repair and complete main/supporting asset area fields.

## Quick Start

Install dependencies first if you are running from source:

```powershell
python -m pip install -r requirements.txt
```

Start the desktop app:

```powershell
python reit_excel_auditor\app.py
```

## Build a Windows Executable

To create a double-clickable Windows executable, run the build script from the project root:

```powershell
python -m pip install -r requirements.txt
.\build_exe.ps1
```

After the build finishes, the executable is generated at:

```text
dist\REITsExcelAuditor.exe
```

Open `dist\REITsExcelAuditor.exe`, choose an input file or folder, choose an output folder, select a conversion mode, optionally choose a metadata workbook or custom template, and click the convert button.

The build process creates these folders:

| Folder | Description |
| --- | --- |
| `dist\` | Contains the final executable. |
| `build\` | Contains PyInstaller intermediate files. |

## Desktop App Workflow

1. Choose an input Excel file or an input folder.
2. Choose an output folder.
3. Select a conversion mode.
4. Optionally choose a metadata workbook for automatic information filling.
5. Optionally enable the Type 4 processed output if you need asset-area repair.
6. Optionally choose a custom output template.
7. Click the convert button.
8. Review the generated workbook and the batch summary workbook in the output folder.

Conversion modes:

| Mode | When to Use |
| --- | --- |
| Auto detect | Use this when you are not sure which built-in type the input belongs to. The tool detects the type from workbook headers. |
| Manual type | Use this when you already know the input workbook type. |
| Custom template | Use this when the input is not one of the built-in types and you want to generate output based on your own template. |

## Output Files

Generated files are saved to the selected output folder.

| File | Description |
| --- | --- |
| `OriginalFileName_自动审核.xlsx` | The converted audit-ready workbook. |
| `OriginalFileName_自动审核_处理版.xlsx` | Generated only for Type 4 input when the processed-output option is enabled. |
| `自动审核_批量汇总.xlsx` | Generated for batch conversion. It records detected type, output paths, warnings, and errors for each input file. |

If an output field cannot be filled automatically, the summary workbook records the field that needs manual review.

The tool identifies workbook types mainly from header fields, not just from file names.

## Optional Type 4 Processed Output

The Type 4 processed workbook is not generated by default. It is created only when you enable the Type 4 processed-output option in the app, or when you use the CLI with `--property-processed`.

The generated file name is:

```text
OriginalFileName_自动审核_处理版.xlsx
```

This workbook is not the standardized summary output. It is a repaired copy based on the original Type 4 detail workbook, designed for cases where asset-area fields may be missing, duplicated, or placed in the wrong column.

Processing logic summary:

- Groups rows by `REITs代码` and `基础设施项目名称`.
- If a project has only a main asset and no supporting asset, the main asset row is marked as `未分类`.
- If total rentable area values differ within the same group, the tool treats them as possible single-asset values placed in the total column, moves them back, and recalculates the total.
- If main and supporting asset single areas are different, the tool sums them and writes the same total back to every row in the group.
- If main and supporting asset single areas are exactly the same, the tool treats them as possible duplicate extraction and keeps the total while clearing duplicated single-area values.
- The same logic is applied to actual rented area fields.
- The processed workbook also normalizes style, filter range, and worksheet view settings.

If the Type 4 source workbook has already been manually verified, you can leave this option disabled and use only the standard output workbook.

## Metadata Workbook

The metadata workbook is used to fill common REITs information into output workbooks. A sample file is provided at:

```text
examples\metadata_template.xlsx
```

Supported fields:

| Field | Description |
| --- | --- |
| `REITs代码` | Used to match source rows with metadata rows. Recommended. |
| `REITs名称` | Used to fill the REITs name. |
| `上市日期` | Used to fill the listing date. |
| `公告日期` | Used to fill announcement date or `INFOPUBLDATE`. |
| `开始日期` | Used to fill `STARTDATE` or start-date fields. |
| `结束日期` | Used to fill `ENDDATE` or end-date fields. |

Not every field is required. The tool uses the fields that are available and reports missing information in the summary workbook.

## Custom Template Output

Custom template output is useful when the input workbook is not one of the built-in Type 1-5 formats, but you already have an Excel template that defines the desired output columns and style.

Good use cases:

- The input workbook is not covered by the built-in conversion rules.
- You only need to extract selected fields into your own template.
- You want the output to inherit your template's column widths, fonts, borders, date formats, percentage formats, number formats, or formulas.
- You have multiple source workbooks with the same structure and want to process them in one folder.

Not recommended for:

- Type 4 area-repair logic. Use the built-in Type 4 mode and enable processed output instead.
- Folders that contain mixed source workbook formats.
- Templates where most headers are merged cells and individual output columns do not have clear field names.

### How to Prepare a Custom Template

1. Create an `.xlsx` workbook with the headers you want in the output.
2. Put sample formatting in the row below the header row, such as date, percentage, currency, decimal places, font, border, or formulas.
3. In the app, select one input workbook or a folder containing workbooks with the same source format.
4. Choose the custom template workbook.
5. Optionally choose a metadata workbook.
6. Start conversion.

### Template Design Tips

- Keep headers on one row when possible. The tool searches the first 20 rows for the most likely header row.
- Avoid merged header cells. If merged cells are necessary, keep a clear field name for each output column.
- Use header names that are close to the source workbook field names.
- Use the row below the header as the style reference row.
- If a column should be calculated by formula, write the formula in the style reference row, such as `=A2+B2`. The tool copies it down as `=A3+B3`, `=A4+B4`, and so on.
- If a template column has no matching source field and no formula, the output cell stays blank and the summary workbook reports it.

### What Custom Templates Handle Automatically

| Capability | Description |
| --- | --- |
| Field order | Output columns follow the template header order. |
| Field matching | The tool tries exact names, aliases, normalized names, containment matching, and similar field names. |
| Style inheritance | The output inherits column width, fonts, borders, alignment, filters, frozen panes, and number formats from the template. |
| Date handling | Date-like columns and fields such as `公告日期`, `STARTDATE`, and `ENDDATE` are converted to dates or 8-digit date numbers when appropriate. |
| Percentage handling | Percentage-format columns and rate-like fields are processed as percentages while preserving real values above 100%. |
| Number handling | Amount, area, count, revenue, cost, profit, asset, and liability fields are converted to numbers when possible. |
| Metadata filling | If a required template field is not in the source workbook, the tool can fill it from the metadata workbook by REITs code. |
| Formula columns | Formulas in the style reference row are copied down and shifted by row number. |

### Common Field Aliases

These aliases can be recognized without manually changing source headers:

| Target Field | Common Aliases |
| --- | --- |
| `REITs代码` | `ReitsCode`, `代码`, `证券代码`, `基金代码`, `产品代码` |
| `REITs名称` | `REITs简称`, `基金名称`, `基金简称`, `证券简称`, `证券名称`, `产品名称` |
| `公告日期` | `INFOPUBLDATE`, `信息披露日期`, `披露日期` |
| `STARTDATE` | `开始日期`, `报告期开始日期`, `报告期起始日期`, `区间开始日期` |
| `ENDDATE` | `结束日期`, `报告期结束日期`, `报告期截止日期`, `区间结束日期` |

You can add more aliases in `config\field_aliases.json`.

### Folder Batch Reminder for Custom Templates

When using a custom template with a folder input, make sure all source workbooks in that folder have the same structure. The tool compares each workbook against the first input workbook. If the headers differ too much, the file is skipped and the reason is written to the summary workbook.

Custom template mode does not run the dedicated built-in repair rules for Type 1-5 tables. For example, Type 4 processed area repair is available only in the built-in Type 4 flow.

## Advanced Configuration

The `config/` folder contains template mapping and field-alias settings.

| File | Purpose |
| --- | --- |
| `config\table_templates.json` | Maps each built-in table type to its standard template workbook. |
| `config\field_aliases.json` | Defines field aliases to improve source-to-target matching. |

Most users do not need to edit these files. Update them only when you rename standard templates or need to add more field aliases.

See:

```text
config\README.md
```

## Command-Line Usage

The desktop app is the recommended interface, but the project also supports command-line conversion:

```powershell
python reit_excel_auditor\app.py --convert "D:\path\input.xlsx" --output-dir "D:\path\output"
```

Useful options:

| Option | Description |
| --- | --- |
| `--convert` | Input Excel file or input folder. |
| `--type` | Conversion type. Defaults to `auto`. |
| `--metadata` | Optional metadata workbook path. |
| `--custom-template` | Optional custom template workbook path. |
| `--output-dir` | Optional output folder. |
| `--property-processed` | Generate the optional Type 4 processed workbook. |

## Troubleshooting

| Problem | Suggested Action |
| --- | --- |
| Missing-field warning | Check whether the source workbook contains the required column, or fill the output manually. |
| Auto detection failed | Try selecting the table type manually, or use custom template output. |
| Some files failed in batch mode | Open `自动审核_批量汇总.xlsx` and review the failure reason. |
| Custom template fields are blank | Make template headers closer to source headers, or add aliases in `config\field_aliases.json`. |
| Excel workbook cannot be opened | Make sure the file is not open in Excel and is not a temporary file starting with `~$`. |
